About gama — EyeLink EDF Explorer

gama opens SR Research EyeLink .EDF recordings and shows them the way SR's own edf2asc would convert them — as event lines (fixations, saccades, blinks, messages) — but in a filterable, spreadsheet-like table. Nothing is modified: the file on disk is only read. Exporting everything with no filters reproduces a byte-identical ASC.

Files & tabs

Each open recording gets a tab. + opens the file browser to add more; ✕ on a tab closes that file. Filter settings are shared across tabs, so you can set up a view once and click between files to compare them.

Presets

All
Clear every filter and show all lines.
Events only
Fixations, saccades and blinks; hides messages and setup.
Messages only
Just MSG lines (your experiment's own markers).
Hide setup
Everything except the file preamble.
Saved presets
Store the current filters (including timestamp mode and visible columns) under a name and reload it later. They are saved as JSON files in the presets folder next to gama, so they persist and can be shared. ✕ deletes one.

Categories

Show or hide whole classes of line. Counts are for the current file.

PREAMBLE
File header written by the tracker (recording date, version).
HEADER
Per-recording START block: eye tracked, sample rate, tracking mode.
END
End of a recording block, with its average screen resolution.
INPUT
Parallel-port / trigger values received by the tracker.
MSG
Messages sent to the tracker (usually by your experiment script).
FIX
Fixations: SFIX (start) and EFIX (end, with the summary values).
SACC
Saccades: SSACC (start) and ESACC (end, with amplitude/velocity).
BLINK
Blinks: SBLINK / EBLINK. Blinks inside one saccade are merged, as edf2asc does.

Message kinds

MSG lines are auto-sorted so tracker chatter can be hidden separately from your own markers.

experiment
Your script's messages (trial IDs, conditions, triggers).
config
Tracker setup: RECCFG, GAZE_COORDS, thresholds, etc.
cal
Calibration and validation results (!CAL, VALIDATE).
draw
DataViewer drawing commands (!V lines: IAs, images, regions).

Filters

Message filter
contains keeps only MSG lines matching the text; excludes drops matching ones. Both apply to the message body only, and only to MSG lines — events are unaffected. Tick regex to use a regular expression instead of plain text.
Eye
Keep only right- or left-eye events. Lines with no eye (messages, inputs) always stay.
Timestamps
Absolute shows raw tracker time (ms since the tracker started). Relative to file start re-bases so the file's first timestamp is 0 — easier to read and to compare across files. This affects the table and CSV/TSV exports; ASC exports always keep absolute times so they stay valid ASC.
Time range
Keep lines whose Start falls between min and max. Uses whichever timestamp mode is selected; the hint underneath shows the available range.
Min duration
Separate thresholds in ms: fixation filters EFIX lines, saccade filters ESACC lines. Rows without a duration (messages, start markers) are never removed by these.
Global search
Free-text search across the raw text of every line, whatever its type. Tick regex for a pattern. Filter rows hides everything that doesn't match; Highlight in place keeps the full view and marks the matches instead, so you can see them in context — use ‹/›, Enter or F3 to step through them.
Appearance
Dark or light theme and three text sizes. Remembered in this browser; not part of a preset.

Selecting & copying

Click a row to select it; shift-click for a range, Ctrl/⌘-click to add or remove one, Ctrl+A for everything in the current view. Ctrl+C copies the selection as tab-separated text with the visible columns — paste straight into Excel. With nothing selected it copies the whole view.

Columns

Use Columns ▾ (top right) to show or hide any of these; click a header to sort. Blank cells simply mean the field doesn't apply to that line type.

#
Line number in the converted file — the original order.
Type
The exact line type (EFIX, ESACC, MSG, START…), coloured by category.
Start
When the line/event began, in tracker ms.
End
When the event ended (end events only).
Dur
Duration in ms (End − Start + 1).
Eye
R or L — the eye the event belongs to.
X1, Y1
For EFIX: the fixation's average gaze position. For ESACC: the saccade's start position. In screen pixels.
X2, Y2
The saccade's end position (ESACC only).
Amp
Saccade amplitude in degrees of visual angle. A value like 3.2e+006 with . in the position columns means gaze data was missing (eye lost) for that saccade.
PVel
Peak velocity of the saccade, in degrees/second.
Pupil
Average pupil size during a fixation (arbitrary units).
ResX, ResY
Screen resolution in pixels per degree, from END lines (hidden by default).
In
The value on an INPUT (trigger) line (hidden by default).
Message / Detail
The message body for MSG lines, or the raw text for preamble/header lines.

Click any row to open a detail pane showing every field plus the exact original line (tabs shown as →).

Exporting

Export ▾ writes out exactly what you are looking at — the current filters and sort.

This file
The active tab, as ASC (real ASC line text), CSV/TSV (the parsed columns above, for Excel/R/Python), or HTML — a self-contained page you can keep or print as a record of the view.
All loaded files
Applies the current filters to every open tab and returns one ZIP with a file per recording.
Copy link to this view
Puts a link on the clipboard that restores these filters, and reopens the same file if the person you send it to can reach that path (a shared drive, say).

Collapsing recording blocks

A recording can contain many START…END blocks (often one per trial). Collapse blocks in the toolbar squeezes each block down to a single summary row — block number, time span, row count and event breakdown. Click a summary (or press Enter / → on it) to expand that block; click the ▼ on a block's first row to re-collapse it. Collapsing is purely visual: it doesn't change what search, copy or export see, and it's automatically suspended while the table is sorted (blocks are only contiguous in the original order).

Flagging & notes

Click a row and use the flag checkbox and note field at the top of the detail pane to mark it — or press F to flag the selected row, or click the small ⚑ in the # column. Flagged rows are tinted and carry the flag marker. Flags and notes are saved to a sidecar file (<name>.EDF.gama-notes.json) right next to the recording, so they travel with the data and reload next time. They're keyed by line number, so they assume the file itself doesn't change. When any row in a file is flagged or noted, CSV/TSV/HTML exports gain flagged and note columns.

Tabs

Double-click a tab to give it a friendlier label than the filename. Labels are remembered in this browser and don't affect exports.
